urllib库

urllib库是python中一个基本的网络请求库。可以模拟浏览器的行为，向指定服务器发送请求，并保存服务器返回数据。

ProxHandler处理器（代理设置）

很多网站会检测某一段时间某个IP的访问次数，访问次数不正常就会限制改IP的访问请求。

  1. 代理的原理：在请求目的服务器之前，先请求代理服务器，然后代理服务器去请求目的服务器，代理服务器获取到网站数据后，将数据转发给本地。

  2. http://httpbin.org 这个网站可以方便的查看http请求的一些参数。

  3. 在代码中使用代理：

    • 使用urllib.request.ProxHandler，传入一个代理，这个代理是一个字典，字典的key依赖于代理服务器能够接受的类型，一般是http或者https，值是ip:port
    • 使用上一步创建的handler,利用request.bulid_opener创建一个opener
    • 使用上一步创建的opener,调用open函数，发起请求。

cookie原理和格式

什么是cookie

在网站中，HTTP是无状态的，也就是说第一次与服务器连接并登陆成功，第二次请求服务依然不能确定当前用户是谁。cookie就是为了解决这个问题而存在的，第一次登陆后服务器会返回一些数据（cookie）给浏览器，然后浏览器会见这些信息存储在本地，当用会再次相同一个服务器发送请求时浏览器会自动将对应服务器的cookie发送给服务器，服务器通过返回的cookie来判断当前请求用户时谁。cookie数据存储量有限，不同的浏览器可能会有不同的大小，但一般不超过4KB，因此使用cookie只能存储一些小量数据。

cookie的格式

  • NAME：cookies的名字。
  • VALUE：cookie的值。
  • Expires：cookie的过期时间。
  • Path：cookie的作用路径。
  • Domain：cookie作用的域名。
  • SECURE：是否只在http协议下起作用。

http.cookiejar模块：

模块主要的类有CookieJar、FileCookieJar、MozillaCookieJar、LWPCookieJAr。四个类的作用分别如下：

  1. CookieJar()：管理http cookie的值，存储http请求过程中生成的cookie，向传出的http请求添加cookie的对象。整个cookie都存储在内存中，对CookieJar实例进行垃圾回收后cookie信息也将丢失。
  2. FileCookieJar(filename,delayload=None,policy=None)：从CookieJar派生而来，用来创建FileCookie实例，检索Cookie的值并将cookie存储在文件中，filename是存储cookie信息的文件名，delayload的值为True时支持文件延时访问，即只有在需要时才读取文件或存入数据。
  3. MozillaCookieJar(filename,delayload=None,policy=None)：从FileCookieJar派生而来，创建与Mozilla浏览器兼容的FilCookieJar实例。
  4. LWPCookieJar(filename,delayload=None,policy=None)：从cookiejar派生而来，创建与libwwwper标准的Set-Cookie3文件格式兼容的FileCookieJar实例。
